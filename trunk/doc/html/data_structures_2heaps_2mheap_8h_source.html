00001 // Header file for a "modulo" heap heap data structure.
00002 // In this heap, keys are 32 bit unsigned integers and
00003 // are ordered circularly around the 32 bit key value
00004 // space and are assumed to never occupy more than half
00005 // the key space. The empty portion of the key space
00006 // effectively defines the ordering of keys. What this
00007 // means is that if a and b are distinct key values, then
00008 // we consider a to be greater than b if (a-b) < 2^31,
00009 // where the subtraction is performed modulo 2^32
00010 // as is standard for unsigned arithmetic.
00011 //
00012 // This data structure is intended for use in applications
00013 // where the key values represent times and these times
00014 // are all clustered around some specific value representing
00015 // the current time. It is up to the application to ensure
00016 // that the items are removed from the heap before their
00017 // key values become too "old" relative to the current time.
00018 // Note that the data structure is oblivious to actual
00019 // time values. It simply assumes that keys are ordered
00020 // as described above.
00021 //
00022 // This heap is also designed so that it can be used as
00023 // either a minheap or a maxheap (but not both at the
00024 // same time). The choice is made by a boolean flag
00025 // passed to the constructor. Note that no error checking
00026 // is done for findmin, deletemin, findmax, deletemax.
00027 // If you call findmin on a max-heap, you will actually
00028 // get the item with the largest key, not the smallest.
