Max Flow Algorithms

This page provides links to source code for various algorithms for the maximum flow problem. There is an associated makefile which can be used to compile all the programs. The makefile assumes a particular directory structure. Specifically, the various groups of algorithms are stored in sub-directories with a common parent. The parent of this directory is assumed to have a child directory called data_structures, where various data_structures can be found.

The links at left provide access to all the same files that can be accessed through the links below. Once one becomes familiar with the various files, these shortcut links provide a more convenient way to access the code.

  • Augmenting Path Algorithm. Implements three variants: max capacity augmentation, capacity scaling and shortest path. Source.
  • Dinic's Algorithm. Source.
  • Dinic's Algorithm with Dynamic Trees. Source.
  • Preflow-push Algorithm. Implements two variants: FIFO with incremental relabeling and FIFO with batch relabeling. Source.
  • Max Flow Command. Reads an input graph and computes a max flow using a specified spt algorithm. Source.
  • Max Flow Repeat Command. Generates random graphs and computes a max flow using a specified algorithm. Source.
  • Max Flow Checker. Reads a flow graph from stdin and checks to see if the defined flow is a legal maximum flow. It prints messages whenever it finds something incorrect. If the flow is correct, there is no output. Usage: check. Source.
